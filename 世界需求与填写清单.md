# AAF 世界需求与填写清单（阶段 8 设计深化版）

本文档用于把“一个世界就是一个完整团务包”的需求冻结下来。

适用范围：单个世界内的业务设计，不含平台级账号、全局大厅等已确认模块。

与开发文档关系：本文件用于需求冻结，进入实施阶段后以 世界细化开发文档.md 作为执行主线。

填写规则：
1. 单选：在选项前加 √（仅一个）。
2. 多选：可在多个选项前加 √。
3. 数值题：在“填写：”后写具体值。
4. 文本题：在“填写：”后写说明。
5. 每项后都填写优先级：P0 必须 / P1 应该 / P2 可选。
6. 每项都尽量给出验收标准，避免只写概念。
