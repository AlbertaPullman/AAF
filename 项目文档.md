一、整体架构
架构模式：前后端分离 + WebSocket实时通信

前端：单页应用（SPA），部署在静态服务器

后端：Node.js API服务 + WebSocket服务，与前端分离部署

数据库：SQLite（初期，文件型，无需额外服务）

通信协议：HTTP（REST API）、WebSocket（Socket.IO）

四、数据库设计（SQLite）
使用Prisma Schema定义数据模型，主要表如下：

User：用户信息（id, username, password, avatar, createdAt, etc.）

Friend：好友关系（userId, friendId, status: PENDING/ACCEPTED）

World：世界/房间（id, name, description, ownerId, visibility: PUBLIC/PASSWORD/FRIENDS/PRIVATE, password（如果加密）, bgmUrl?, createdAt）

WorldMember：世界成员（worldId, userId, role: GM/PLAYER, status: ACTIVE/INVITED）

Scene：场景（id, worldId, name, backgroundImageUrl, gridSize?, data（存储Fabric画布JSON）, createdAt）

Character：角色卡（id, worldId, userId?（如果关联玩家）, name, avatar, type: PC/NPC, stats（JSON）, aiConfig?（用于酒馆NPC）, createdAt）

Message：聊天记录（id, type: GLOBAL/WORLD/PRIVATE, worldId?（如果是世界聊天）, fromUserId, toUserId?（私聊）, content, createdAt）

AiSession：AI酒馆会话（id, worldId, npcId, userId, messages（JSON历史）, createdAt）

五、核心功能模块及技术实现
1. 用户认证与社交
注册/登录：JWT token存储在localStorage或httpOnly Cookie（推荐localStorage简化）

好友系统：基于数据库好友表，前端展示好友列表，可通过Socket.IO发送好友请求通知

聊天系统：

全局聊天室：所有在线用户加入Socket.IO global room

世界聊天室：每个世界对应一个Socket.IO room（如world:${worldId}），成员加入

私聊：可基于Socket.IO私发，需要维护在线用户socket映射（可选，初期可先不做）

2. 世界（房间）管理
创建世界：填写名称、描述、可见性、密码等，后端存入World表，自动将创建者设为GM

加入世界：根据可见性验证，密码世界需提供密码，好友世界需检查好友关系

世界列表：展示所有公开世界，以及用户有权限加入的世界

3. VTT核心功能（基于Fabric.js + Socket.IO）
场景管理：

每个世界可包含多个场景，切换场景时加载对应的Fabric画布JSON

场景背景：上传图片，作为画布背景

Token：

代表角色或怪物，可移动、旋转、缩放

拖拽时通过Socket.IO广播位置更新，其他玩家实时同步

测量尺：

Fabric画布上绘制线段，显示距离（基于格子尺寸或像素比例）

绘制工具：

画笔、擦除、绘制矩形/圆形等，使用Fabric对象实现

BGM：

上传音频文件，后端存储，前端用Howler.js播放，支持循环、音量控制

场地特效：

如动态光效、粒子效果，可用Fabric的滤镜或叠加图层实现（可选，后期）

4. AI酒馆（智能NPC）
配置：GM在角色卡中填写NPC的性格、背景等，存入aiConfig字段（JSON）

交互：

玩家在“酒馆”界面选择NPC，发送消息

后端接收消息，构造Prompt，调用大模型API（如智谱GLM-4-Flash免费版）

将模型回复返回给玩家，并保存对话历史

API集成：

使用 axios 调用智谱API，需在后端配置API Key（环境变量）

注意限流和成本控制（限制每日对话次数）

5. GM助手
功能：GM在描述场景时，输入关键词（如“山谷”），AI生成一段描述文本

实现：后端提供接口 /api/gm/describe，接收关键词和上下文，调用模型返回文本

前端：在GM界面增加浮动按钮，点击弹出输入框，生成后插入聊天框

项目概述：自定义TRPG虚拟桌面平台
一、项目背景与目标
本项目旨在为我和朋友们打造一个专属的线上跑团（TRPG）平台。由于现有的虚拟桌面（如Foundry VTT）无法完美支持我们自创的融合规则（结合了D&D 5e和“最终无语”的机制），且我们希望能深度集成社交互动与AI辅助叙事，因此决定从零开发一个定制化网站。

平台的核心目标是：

提供一个灵活、可扩展的虚拟桌面，实现跑团所需的基本功能（地图、棋子、骰子、绘制等）。

构建一个轻量级的社交系统，让朋友们可以方便地创建跑团房间（世界）、管理好友关系、跨房间聊天。

引入AI能力，实现智能NPC（酒馆系统）和GM创作辅助，提升跑团的沉浸感与便利性。

所有功能都围绕我们自定的规则体系设计，但保持足够的通用性，未来可适配其他规则。

二、用户角色
主持人（GM）：创建和管理世界（跑团房间），配置场景、NPC、规则，主持游戏进程。

玩家（Player）：加入世界，扮演角色，参与游戏，并可在酒馆中与AI NPC互动。

三、功能模块详解
1. 用户与社交系统
注册/登录：用户通过用户名和密码注册登录，采用JWT鉴权。

好友管理：支持搜索用户、发送好友请求、接受/拒绝请求。好友关系用于世界权限控制（如“仅好友可见”）。

聊天系统：

全局聊天室：所有在线用户可自由交流，分享世界链接。

世界内聊天：每个世界拥有独立的聊天频道，只有世界成员可见。

私聊（可选，后期）：好友之间一对一会话。

用户资料：头像、昵称等基本信息。

2. 世界（跑团房间）管理
创建世界：GM填写世界名称、描述、设置可见性（公开/密码/仅好友/仅自己）、上传封面图、可选设置密码。

加入世界：玩家根据可见性规则申请/直接加入。密码世界需输入密码；好友世界需GM或好友邀请。

世界列表：大厅展示所有可进入的世界，支持筛选（公开、好友的、我加入的）。

世界成员管理：GM可查看成员列表，移除玩家，转让主持权等。

3. 虚拟桌面核心（VTT）
每个世界包含多个场景（如不同地图），GM可以切换场景，玩家同步看到当前场景。

场景管理：

创建/删除场景，设置场景名称、背景图片（上传）。

切换场景时，所有成员画布自动加载新场景。

画布基础功能：

地图背景：支持上传图片作为地图，可缩放、平移。

网格（可选）：可开启网格辅助定位，设置格子大小。

测量尺：拖拽测量两点距离，显示格数或像素。

绘制工具：画笔、线条、矩形、圆形等，用于临时标注（GM专用或玩家可见可选）。

清除绘制：一键清除临时绘制。

棋子（Token）系统：

代表角色、怪物、NPC，可放置在地图上。

每个Token可关联角色卡（见下文），显示名称、血量等状态。

支持拖拽移动（实时同步给其他人），旋转，缩放。

右键菜单：查看详情、编辑属性。

BGM与音效：

GM可上传音频文件，为场景设置背景音乐。

支持播放、暂停、音量调节，循环播放。

可触发即时音效（如战斗音效）。

骰子（基础）：

内置骰子指令（如/d20），掷骰结果在聊天室显示。

可扩展自定义骰子公式（基于我们自创规则）。

4. 角色卡系统
GM可创建角色卡模板（如玩家角色、怪物、NPC）。

角色卡包含：名称、头像、属性（可自定义字段）、描述、备注。

对于AI NPC，GM可额外配置AI设定（性格、说话风格、背景故事），用于酒馆交互。

玩家可查看自己扮演的角色卡，GM可编辑所有角色。

5. AI酒馆（智能NPC）
触发方式：在世界内增加“酒馆”入口，玩家可进入酒馆界面。

NPC列表：展示当前世界中配置了AI设定的所有NPC，玩家选择一位开始对话。

对话机制：

玩家发送消息，后端调用大模型API（如智谱GLM-4-Flash，免费），并将NPC设定、对话历史一起提交。

模型生成的回复返回给玩家，并保存对话记录，形成连续对话。

GM控制：GM可设置每个NPC的对话开关、每日对话次数限制、重置对话历史。

价值：在非跑团时段，玩家可以与NPC互动，深化角色关系，同时也减轻GM扮演NPC的负担。

6. GM助手（AI创作辅助）
场景描述生成：GM在描述场景时，可点击AI助手图标，输入主题（如“幽暗森林”），AI生成一段富有氛围的文本描述。

NPC对白建议：GM在配置NPC时，可让AI根据角色设定生成示例对话。

剧情灵感：GM可请求AI提供冒险点子、谜题、随机遭遇等。

所有AI生成结果均可一键插入聊天框或角色卡备注。

四、关键特性与设计原则
组件化架构：VTT功能拆分为独立组件（画布、工具栏、角色面板、聊天等），便于维护和扩展。

实时同步：所有玩家看到的画布状态（棋子位置、绘制内容）保持一致，通过WebSocket实现低延迟同步。

社交隔离：好友系统确保陌生人不能随意进入私人世界；聊天频道隔离避免信息混杂。

低成本部署：初期使用SQLite文件数据库，低配VPS即可运行，所有AI服务采用免费额度或极低成本的API。

AI优先体验：将AI无缝融入游戏过程，而非外挂工具。
