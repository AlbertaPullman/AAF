# 阶段 7 执行任务清单 - 场景系统、角色卡、世界内聊天

## 1. 当前状态

**✅ 已完成（阶段 7 核心链路收尾完成）** - 2026-03-13

本阶段目标：在阶段 6 最小 VTT 同步基础上，补齐世界内业务能力，包括聊天频道、角色卡关联与场景增强。

## 2. 本轮完成内容（阶段 7 子任务 1）

### 2.1 世界聊天 HTTP 能力

- 新增接口：`GET /api/chat/worlds/:worldId/recent`
- 支持最近消息读取（limit 1-100）。
- 仅世界 ACTIVE 成员可读取。

### 2.2 世界聊天 Socket 事件

- 新增事件：
  - `world:message:send`（client -> server）
  - `world:message:new`（server -> client）
- 仅加入房间且为 ACTIVE 成员时允许发送。
- 服务端写入 `Message(type=WORLD)` 后广播到对应世界 room。

### 2.3 世界页接入

- 世界页新增“世界内聊天”面板。
- 首次加载世界时拉取最近世界消息。
- 接入实时事件监听与消息发送 ack 处理。

## 2.4 本轮完成内容（阶段 7 子任务 2）

### 2.4 角色卡最小读写

- 新增接口：
  - `GET /api/worlds/:worldId/characters`
  - `POST /api/worlds/:worldId/characters`
- ACTIVE 成员可查看角色列表。
- GM 可创建 PC/NPC；非 GM 仅可创建绑定自己的 PC。

### 2.5 Token 与角色绑定

- token move payload 新增 `characterId`。
- token 广播状态新增 `characterId` 与 `characterName`。
- 服务端在同步时校验角色归属与世界归属。
- 非 GM 不可绑定或移动他人角色 token。

### 2.6 世界页角色面板

- 新增 CharacterPanel：角色列表、当前绑定角色选择、新建角色表单。
- 新增我的 token 时默认绑定当前选择角色。

## 2.7 本轮完成内容（阶段 7 子任务 3）

### 2.7 角色详情最小编辑

- 新增接口：`PUT /api/worlds/:worldId/characters/:characterId`。
- 支持编辑：`name`、`stats`、`snapshot`。
- 权限规则：GM 可编辑任意角色，非 GM 仅可编辑自己角色。
- 世界页 CharacterPanel 新增详情编辑区域（JSON 形式）。

## 2.8 本轮完成内容（阶段 7 子任务 4）

### 2.8 世界聊天多频道

- 世界聊天频道扩展为 `OOC`、`IC`、`SYSTEM`。
- 历史接口支持 `channelKey` 过滤。
- Socket 世界聊天收发 payload 新增 `channelKey`。
- 世界页聊天面板新增频道切换并按频道加载历史。

## 2.9 本轮完成内容（阶段 7 子任务 5）

### 2.9 频道权限（SYSTEM）

- SYSTEM 频道发送权限限制为 GM。
- 非 GM 在前端会被禁用发送输入与按钮。
- 服务端增加兜底校验，违规发送返回 `channel permission denied`。
- 新增权限单元测试：`server/src/services/chat.channel.test.ts`。

## 2.10 本轮完成内容（阶段 7 子任务 6）

### 2.10 频道未读计数（最小版）

- 世界页新增 OOC/IC/SYSTEM 三频道未读计数状态。
- 收到非当前频道消息时对应频道未读 +1。
- 切换到频道并加载历史后，当前频道未读清零。

## 2.11 本轮完成内容（阶段 7 子任务 7）

### 2.11 角色详情结构化编辑

- 角色详情编辑从 JSON 文本改为结构化字段：HP、MP、等级、职业。
- 保存时前端自动组装为 `stats` 与 `snapshot`，减少手写 JSON 出错。
- 保留后端原有权限模型（GM 可改任意角色，非 GM 仅改自己角色）。

## 2.12 本轮完成内容（阶段 7 子任务 8）

### 2.12 未读持久化与消息审计（最小版）

- 频道未读计数写入 localStorage，刷新后按世界与用户恢复。
- 服务端新增世界频道消息发送审计日志（world/channel/user/messageId）。

## 2.13 本轮完成内容（阶段 7 子任务 9）

### 2.13 角色权限测试覆盖（单元）

- 抽离角色创建/编辑权限判定函数，便于复用与测试。
- 新增角色权限单元测试：覆盖 GM 与非 GM 的创建和编辑边界。

## 2.14 本轮完成内容（阶段 7 子任务 10）

### 2.14 多 Scene 切换最小链路

- 新增接口：
  - `GET /api/worlds/:worldId/scenes`
  - `POST /api/worlds/:worldId/scenes`
- 新增 Socket 事件：
  - `scene:select`（client -> server）
- 服务端 token 状态缓存由 world 级升级为 scene 级（`worldId:sceneId`）。
- `scene:token:move/moved` payload 增加 `sceneId`，实现场景隔离同步。
- 世界页新增 ScenePanel，支持场景列表、切换与创建（最小版）。
- 已有频道权限单测继续保留，测试总数增加。

## 2.15 本轮完成内容（阶段 7 子任务 11）

### 2.15 多 Scene 聊天上下文隔离（最小版）

- 世界聊天历史接口支持 `sceneId` 参数。
- `world:message:send/new` payload 增加 `sceneId`。
- 服务端使用消息 metadata 记录场景上下文并按 scene 过滤历史。
- 世界页聊天显示、发送与未读持久化改为 world+scene 维度。

## 2.16 本轮完成内容（阶段 7 子任务 12）

### 2.16 场景重命名（最小版）

- 新增接口：`PUT /api/worlds/:worldId/scenes/:sceneId`。
- 仅 GM 可重命名场景，且校验 scene 必须属于当前 world。
- 世界页 ScenePanel 新增“重命名当前场景”输入与保存按钮。

## 2.17 本轮完成内容（阶段 7 子任务 13）

### 2.17 场景删除（最小版）

- 新增接口：`DELETE /api/worlds/:worldId/scenes/:sceneId`。
- 仅 GM 可删除场景，且至少保留 1 个场景。
- 世界页 ScenePanel 新增“删除当前场景”按钮与基础保护。

## 2.18 本轮完成内容（阶段 7 子任务 14）

### 2.18 场景排序（最小版）

- 支持 `direction=UP|DOWN` 场景顺序调整。
- 服务端采用事务交换 `sortOrder`，避免冲突。
- 世界页 ScenePanel 新增上移/下移按钮。

## 4. 验收点

1. 世界成员进入世界页后可查看最近世界聊天历史。
2. 用户发送消息后，同世界在线成员可实时收到 `world:message:new`。
3. 非成员或未 join 房间时发送会返回错误 ack。
4. 场景切换后 token 与世界聊天都能按 scene 正确隔离。
5. GM 可执行场景创建、重命名、删除、排序，非 GM 会被拒绝。
6. 关键规则与链路已具备单元测试、HTTP 测试与 Socket 集成测试覆盖。
