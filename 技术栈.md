FVTT风格TRPG平台技术栈文档
本文档用于指导基于AI的全栈开发，所有技术选型均考虑零成本、易用性、AI友好。请将本文档提供给AI，以便生成项目骨架代码。

一、整体架构
架构模式：前后端分离 + WebSocket实时通信

前端：单页应用（SPA），部署在静态服务器

后端：Node.js API服务 + WebSocket服务，与前端分离部署

数据库：SQLite（初期，文件型，无需额外服务）

通信协议：HTTP（REST API）、WebSocket（Socket.IO）

四、数据库设计（SQLite）
使用Prisma Schema定义数据模型，主要表如下：

User：用户信息（id, username, password, avatar, createdAt, etc.）

Friend：好友关系（userId, friendId, status: PENDING/ACCEPTED）

World：世界/房间（id, name, description, ownerId, visibility: PUBLIC/PASSWORD/FRIENDS/PRIVATE, password（如果加密）, bgmUrl?, createdAt）

WorldMember：世界成员（worldId, userId, role: GM/PLAYER, status: ACTIVE/INVITED）

Scene：场景（id, worldId, name, backgroundImageUrl, gridSize?, data（存储Fabric画布JSON）, createdAt）

Character：角色卡（id, worldId, userId?（如果关联玩家）, name, avatar, type: PC/NPC, stats（JSON）, aiConfig?（用于酒馆NPC）, createdAt）

Message：聊天记录（id, type: GLOBAL/WORLD/PRIVATE, worldId?（如果是世界聊天）, fromUserId, toUserId?（私聊）, content, createdAt）

AiSession：AI酒馆会话（id, worldId, npcId, userId, messages（JSON历史）, createdAt）

五、核心功能模块及技术实现
1. 用户认证与社交
注册/登录：JWT token存储在localStorage或httpOnly Cookie（推荐localStorage简化）

好友系统：基于数据库好友表，前端展示好友列表，可通过Socket.IO发送好友请求通知

聊天系统：

全局聊天室：所有在线用户加入Socket.IO global room

世界聊天室：每个世界对应一个Socket.IO room（如world:${worldId}），成员加入

私聊：可基于Socket.IO私发，需要维护在线用户socket映射（可选，初期可先不做）

2. 世界（房间）管理
创建世界：填写名称、描述、可见性、密码等，后端存入World表，自动将创建者设为GM

加入世界：根据可见性验证，密码世界需提供密码，好友世界需检查好友关系

世界列表：展示所有公开世界，以及用户有权限加入的世界

3. VTT核心功能（基于Fabric.js + Socket.IO）
场景管理：

每个世界可包含多个场景，切换场景时加载对应的Fabric画布JSON

场景背景：上传图片，作为画布背景

Token：

代表角色或怪物，可移动、旋转、缩放

拖拽时通过Socket.IO广播位置更新，其他玩家实时同步

测量尺：

Fabric画布上绘制线段，显示距离（基于格子尺寸或像素比例）

绘制工具：

画笔、擦除、绘制矩形/圆形等，使用Fabric对象实现

BGM：

上传音频文件，后端存储，前端用Howler.js播放，支持循环、音量控制

场地特效：

如动态光效、粒子效果，可用Fabric的滤镜或叠加图层实现（可选，后期）

4. AI酒馆（智能NPC）
配置：GM在角色卡中填写NPC的性格、背景等，存入aiConfig字段（JSON）

交互：

玩家在“酒馆”界面选择NPC，发送消息

后端接收消息，构造Prompt，调用大模型API（如智谱GLM-4-Flash免费版）

将模型回复返回给玩家，并保存对话历史

API集成：

使用 axios 调用智谱API，需在后端配置API Key（环境变量）

注意限流和成本控制（限制每日对话次数）

5. GM助手
功能：GM在描述场景时，输入关键词（如“山谷”），AI生成一段描述文本

实现：后端提供接口 /api/gm/describe，接收关键词和上下文，调用模型返回文本

前端：在GM界面增加浮动按钮，点击弹出输入框，生成后插入聊天框

七、开发流程建议（AI协作）
初始化项目：使用AI生成前后端基础结构（create-react-app + express-generator）。

模块化开发：按功能模块拆分任务（如用户模块、世界模块、VTT画布），每个模块给出详细提示词。

迭代测试：每完成一个小功能，手动测试，将错误反馈给AI修复。

数据持久化：使用Prisma迁移工具生成数据库表。

部署：按AI生成的部署指南配置服务器。

八、注意事项
环境变量：所有敏感信息（JWT密钥、API密钥）存放在 .env 文件，不提交到Git。

安全性：使用JWT过期时间、密码加密、接口限流。

错误处理：前后端统一错误格式，便于调试。

性能：VTT画布同步时，只传输必要数据（如移动坐标），避免发送整个画布JSON。
