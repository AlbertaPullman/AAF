# 技术栈

## 总体架构

- 架构模式：前后端分离 + HTTP API + Socket.IO 实时通信
- 目标场景：中文 TRPG 世界管理、场景协作、角色运行时、规则与世界扩展
- 数据库：SQLite
- ORM：Prisma

## 前端

- React 18
- TypeScript
- Vite
- React Router DOM
- Axios
- Zustand
- Socket.IO Client
- Lucide React
- Quill / React Quill
- X6

前端职责：

- 账号与大厅界面
- 世界页与场景交互
- 实体管理、规则编辑、天赋试用等管理界面
- 实时聊天与场景同步展示

## 后端

- Node.js
- Express
- Socket.IO
- Prisma
- bcrypt
- jsonwebtoken
- helmet
- cors
- morgan

后端职责：

- 用户认证与权限控制
- 世界、场景、角色、聊天、规则等业务接口
- WebSocket 房间管理与实时广播
- 规则、结算、运行时模块等服务逻辑

## 数据与共享层

- `server/prisma/schema.prisma`：数据库模型
- `shared/constants`：共享常量
- `shared/types`：共享类型
- `shared/rules`：前后端共用规则函数

当前约定：

- 共享规则优先抽到 `shared/`
- 纯规则函数优先保持无副作用
- 前端不直接依赖服务端内部实现

## 开发命令

```bash
npm run dev
npm run build
npm run db:generate
npm run db:migrate
npm run db:seed
npm test -w server
```

## 当前技术判断

- SQLite + Prisma 适合当前单机与小规模协作阶段
- React + Vite 足够支撑当前复杂管理页与实时交互
- Socket.IO 继续作为世界内实时同步主通道
- 后续若进入正式部署与更高并发，再评估数据库、缓存和静态资源拆分
