# 角色卡设计文档

## 1. 文档目标

- 用途：统一记录角色卡的交互规范、字段结构、接口契约与迭代决策。
- 规则：每次涉及角色卡开发（大厅或世界内）都同步更新本文件。
- 目标：支持“一站式车卡 -> 后续自动化战斗接入”的渐进式落地。

## 2. 当前范围（2026-03-26）

- 当前只做：一站式半自动车卡骨架。
- 暂不做：世界内自动化战斗执行逻辑。
- 大厅与世界内角色卡：采用同一组件骨架，避免双套 UI/逻辑分叉。

## 3. 已落地 UI/交互规范

### 3.1 弹窗与层级

- 弹窗尺寸：约为天赋树预览弹窗的一半。
- 默认位置：屏幕中心。
- 拖拽：长按顶部栏可拖拽移动。
- 背景处理：
  - 不做背景失焦/变暗。
  - 不支持点击弹窗外自动关闭（仅右上角关闭按钮关闭）。

### 3.2 顶部栏按钮（右上图标按钮）

- 按钮顺序（从右到左）：
  - 关闭角色卡（红叉）
  - Token 配置（齿轮）
  - 导出角色卡
  - 导入角色卡
- 展示要求：仅图标，无文字。

### 3.3 主次区域布局

- 主数据区域：弹窗上部约 30%。
- 次数据区域：弹窗下部主要内容区。
- 右侧书签页签：用于切换次要区域内容。
- 页签清单：详情、装备、特性、法术、战技、状态、天赋、副职、传记。
- 页签展示：默认仅图标（或图标化缩写），悬停显示文本。

## 4. 角色数据字段草案（第一版）

> 说明：字段先按车卡所需最小集合定义，后续战斗自动化接入时扩展。

### 4.1 基础身份字段

- `id: string`
- `worldId?: string`
- `name: string`
- `avatarUrl?: string`
- `playerUserId?: string`
- `alignment?: string`
- `race?: string`
- `background?: string`
- `classPrimary?: string`
- `subclass?: string`
- `level: number`

### 4.2 核心数值字段

- `attributes: { str: number; dex: number; con: number; int: number; wis: number; cha: number }`
- `hp: { current: number; max: number }`
- `resourcePools?: Record<string, { current: number; max: number }>`
- `armorClass?: number`
- `initiativeBonus?: number`
- `movement?: number`

### 4.3 次要结构字段

- `equipment: CharacterEquipmentItem[]`
- `traits: CharacterTraitItem[]`
- `spells: CharacterSpellItem[]`
- `techniques: CharacterTechniqueItem[]`
- `statuses: CharacterStatusItem[]`
- `talentAllocation?: CharacterTalentAllocation`
- `biography?: { summary?: string; milestones?: string[]; notes?: string }`

## 5. 接口草案（待后端对齐）

- `GET /api/character-sheets/:id/export`（下载 JSON）

## 6. 与天赋系统的硬性约束

  - `parseAffixMeta(value, rawMeta)`
  - `canUnlockNode(node, ranks, professionLevels, allNodes)`
  - `canLearnNode(node, ranks, committedRanks, points, allNodes)`

## 12. 2026-03-26 预创建启动流程（基础版）

- 初次点击“角色卡预创建”时，先弹出“是否需要车卡教程”的询问弹窗：
  - 选择“是，需要教程”：记录用户意向后继续进入快速创建。
  - 选择“否，直接创建”：直接进入快速创建。
- 新手车卡教程本体暂不实现，当前仅保留入口与流程占位。
- 快速创建弹窗当前字段：
  - 已启用：角色名字（必填）。
  - 预留未启用：角色等级（暂不开放）。
- 用户确认快速创建后，再打开角色卡主体窗口（而不是直接打开主体窗口）。
- 角色卡顶部左侧标题显示为“角色名字”而非“角色卡预创建”。
- 世界内角色卡后续沿用同一逻辑，但本阶段仅实现大厅预创建流程。

  - 开关关闭时为只读状态，开启后才允许编辑。
  - 当前已在“主要数据区域”和“次要数据区域”放置该开关。
- 主要数据区左侧：
  - 移除立绘圆框、周边按钮、绿色生命横条。
  - 改为两个垂直波动条：生命值、魔力值。
  - 在生命/魔力下方新增“临时生命值”圆角矩形框。
- 主要数据区右侧上半部三行：
  - 第 1 行：角色名称 / 经验值 / 等级 / 编辑模式滑块。
  - 第 2 行：体型、生物类型（左）/ 熟练加值（右）。
  - 第 3 行：速度（左）/ 专注加值（右）。

  - 调整为两行：
    - 上行：六大属性模块（力量、敏捷、体质、智力、感知、魅力）。
    - 下行：六项次级数值（AC、先攻、战意值、技力、法术强度、物语点）。

- 数据状态说明（当前阶段）：
  - 生命值、魔力值、经验值、等级、速度、AC、先攻、六维属性均为写死占位值。
  - 自动化计算逻辑暂未接入，后续在规则引擎与数值管线完成后替换为实时计算值。

  - 技能点列表弹层当前留空占位，后续接入真实技能点数据。

  - 最右：被动检定值（不带符号），计算规则为 `10 + 调整值`。

## 18. 2026-04-02 装备页与物品页签骨架（第一版）

- 页签新增：
  - 在“详情”与“装备”之间新增“职业”页签。
  - 在“装备”页签下方新增“物品”页签。
  - 语义分工：
    - 职业页：用于展示角色拥有的职业与对应等级。
    - 装备页：用于装备各类物品与道具（穿戴、手持、挂载）。
    - 物品页：用于背包/仓库式存放与整理。
- 职业页布局（方框列表）：
  - 主体为若干职业方框，每个职业占据一个方框。
  - 方框展示字段：职业名 + `X级`（该职业等级）。
  - 当玩家未拥有该职业等级时，默认显示 `0级`。
  - 当前职业清单：战士、狂怒斗士、影刃、秘武者、骑士、祭司、猎魔人、灵语者、吟游诗人、魔法师、魔能使、机兵士。
- 物品页布局（骨架版）：
  - 顶部为分类切换栏（先占位，后续可调整）：武器、施法、消耗品、魔法物品、冒险装备。
  - 切换分类后，主体区域展示该分类对应条目。
  - 主体为左右两栏同构列表，视觉结构一致。
  - 条目自动排布规则：新增条目按“左 -> 右”顺序依次落位（第1条左栏、第2条右栏、第3条左栏……）。
  - 本阶段仅做展示，不接入手动添加入口。
- 特性页布局（与物品页同构）：
  - 顶部分类切换：种族、职业、背景、命途、技能。
  - 切换分类后，主体区域按左右两栏同构展示特性条目。
  - 条目排布规则与物品页一致：按“左 -> 右”顺序依次落位。
- 法术页布局（与物品页同构）：
  - 顶部分类切换：戏法、初级、中级、高级、史诗、传说、仪式。
  - 切换分类后，主体区域按左右两栏同构展示法术条目。
  - 条目排布规则与物品页一致：按“左 -> 右”顺序依次落位。
- 战技页布局（与物品页同构）：
  - 顶部分类切换：初传、进阶、化境、登峰、无相。
  - 切换分类后，主体区域按左右两栏同构展示战技条目。
  - 条目排布规则与物品页一致：按“左 -> 右”顺序依次落位。
- 天赋页布局（三栏骨架）：
  - 页面主体按三栏布局：左侧通用天赋树，中间职业天赋树，右侧技能点。
  - 每栏顶部显示点数统计：已占用点数 / 总点数。
    - 通用与职业天赋点独立计算。
    - 技能栏使用技能点统计。
  - 总点数定义：角色当前拥有的总点数（包含已占用与未占用）。
  - 已占用点数定义：当前已消耗在对应栏位树中的点数总和。
  - 每栏初始包含一个空槽：
    - 通用/职业：`学习新天赋`
    - 技能：`升级技能`
  - 点击空槽后流程：
    - 第一步：弹出对应栏位的可选天赋树列表。
    - 第二步：单击列表项后弹出“学习天赋树/升级技能”确认窗口。
  - 学习后显示天赋槽信息：
    - 第一行：天赋树名称。
    - 第二行：`已消耗：XX点`。
    - 第三行：`解锁节点：XX/XX`（已解锁节点数 / 总节点数）。
- 状态页布局（四分区骨架）：
  - 页面自上而下固定分为四个区域：增益、减益、效果、状态。
  - 增益与减益：战场临时类，来源会随能力变化而扩展，当前先留空。
  - 效果与状态：固定分类，当前先留空，后续按固定条目接入。
- 装备页整体结构：
  - 次要区域改为左右 `1:1` 对半布局。
  - 装备页内部不再显示顶部“装备”标题行，直接把空间留给左右内容区。
  - 右侧改为“可折叠快速查看区”：
    - 顺序固定为：武器、法器、装备。
    - 每个分组均支持收起/展开。
    - 玩家可将条目手动添加到对应分组，也可从分组中移除。
    - 条目显示最小信息：伤害、使用方式、数量。
    - 对已装备条目显示“已装备”标记；双持语境下可显示“左手/右手/双手”标记。
- 装备页左侧布局（参考 MMORPG 装备界面）：
  - 中央：人物展示区（立绘/模型占位），顶部区域保留给人物展示。
  - 周围：方形装备槽按左右两列环绕排布（顶部不放装备槽）。
  - 已落地槽位：
    - 左列（由上到下）：头部、颈部、披肩、衣袍、盔甲、腰带。
    - 右列（由上到下）：护腕、脚部、戒指①、戒指②、饰品①、饰品②。
  - 双手槽位：
    - 左手、右手槽位放置在环绕装备槽区域下方。
- 宝物槽规范：
  - 放置在双手槽位下方，与双手槽位仅保留少量间距。
  - 不再使用外层容器边框，直接以槽位网格呈现，减少占用空间。
  - 默认显示 2 个槽位。
  - 预留扩展位与可扩展排布空间，后续可继续增加宝物槽数量。
- 本阶段未实现项（已记录）：
  - 装备条目的快速槽位切换（例如左手/右手一键互换）暂不实现，后续迭代补充。
  - 武器栏“添加装备”将改为公共按钮：点击后弹出统一选择弹窗，数据源包含“角色背包”和“团队据点仓库”，支持批量勾选后一次加入关注栏；本阶段先记录方案，不实现交互。
  - 物品页手动添加/移除条目暂不实现，后续和背包、仓库数据源打通后再接入。
  - 特性/法术/战技页手动添加、编辑、移除条目暂不实现，后续与角色数据源联动时接入。
  - 天赋页暂不接入真实点数结算、节点学习规则与回退逻辑，当前仅实现展示级骨架与交互流程。
  - 状态页条目填充、来源联动与持续时间结算暂不实现，后续在战斗系统联调阶段接入。

## 19. 2026-04-08 角色卡第一轮实装（导入导出 + 设置菜单 + 技能编辑）

- 导入导出（JSON）：
  - 角色卡支持导出为 `.json`，文件命名规则固定为：`角色名_人物等级_玩家名_年月日_时分秒.json`。
  - 导出时间戳格式示例：`2026年04月08日_153045`。
  - 导入支持同格式 JSON 回填当前角色卡（技能配置、Token、名称、等级）。
  - 导出结构包含：`source`、`schemaVersion`、`characterName`、`characterLevel`、`playerName`、`proficiencyBonus`、`tokenDataUrl`、`skills[]`。
  - 本地缓存与导出共用同一套 payload 结构，避免双标准分叉。

- JSON 结构维护硬规则（防遗忘）：
  - 后续角色卡字段新增或删减时，必须同步执行 3 件事：
    - 更新 `CharacterSheetWorkbench.tsx` 中的 `CharacterSheetExportPayload` 类型。
    - 更新导入校验函数 `isCharacterSheetExportPayload` 与导入映射逻辑。
    - 递增 `schemaVersion` 并同步更新本设计文档对应条目。
  - 任何一次字段变更不得只改 UI 不改导入导出结构，避免旧档与新档错位。

- 右上角按钮改造：
  - 原“Token 配置”按钮改为“设置”按钮（齿轮图标）。
  - 点击后显示小型下拉列表，第一项为“编辑模式”开关（开启/关闭）。
  - 设置下拉内保留 Token 设置入口，后续集中挂载更多配置项。

- 编辑模式规则：
  - 默认只读。
  - 编辑模式开启后，角色卡出现明显视觉态变化（边框与高亮阴影）。
  - 涉及字段改动的操作（如上传 Token、技能配置弹窗）仅允许在编辑模式执行。

- 技能栏实装：
  - 移除技能行中的“属性无”显示列。
  - 顶部新增标题行：`熟练项` / `技能` / `技能加值` / `被动检定`。
  - 编辑模式下可点击技能名称打开配置弹窗。
  - 弹窗字段：
    - 关联属性（无/六大属性）。
    - 熟练项三选一（无/熟练/专精）。
    - 常驻调整值（可输入文本）。
    - 临时调整值（可输入文本）。
  - 技能加值与被动检定支持悬停来源提示（属性调整值、熟练加值、无来源输入）。

- Ability System 前瞻记录（仅记录，不在本阶段开发）：
  - 未来将增加“公式索引插件”：输入公式时弹出可选公式/数值清单，支持单击插入。
  - 公式条目需要提供中文用途说明，便于快速理解与配置。

### 20.9 隐私字段标记

每个字段附 `hiddenFromPlayers: boolean`（在 GM 编辑怪物角色卡时配置）。

**默认隐藏列表**（GM 在怪物模板上勾选时自动应用）：

- HP（current/max/temporary）
- MP（current/max）
- 战意值
- 技力
- AC
- 抗性 / 免疫 / 易伤
- 状态免疫
- 苦难免疫
- 多职业（professions 数组）
- **怪物能力**（怪物的特性，而非职业特性，默认对玩家隐藏）

其余字段默认可见，由 GM 视情况调整。

### 20.10 v2 → v3 导入映射

```
v2.characterName       → v3.identity.name
v2.characterLevel      → v3.identity.totalLevel + 单职业 fallback：professions = [{ professionId: v2.classPrimary || 'unknown', level: v2.characterLevel }]
v2.playerName          → 通过 playerUserId 反查
v2.proficiencyBonus    → v3.stats.proficiencyBonus
v2.tokenDataUrl        → v3.config.tokenConfig.currentDataUrl
v2.skills[]            → v3.business.skills（沿用现有结构）

不存在于 v2 的字段：
- v3.stats.combatWill / techniquePoints / spellPower / storyPoints / casterLevels：默认 { current: 0, max: 0 } 或空对象
- v3.identity.size / creatureType：默认 undefined（UI 提示填写）
- v3.defense.senses / resistance：默认空对象
- v3.config.reactionPresets / intelligenceLog：默认空
- v3.business.sufferings：默认空数组
```
