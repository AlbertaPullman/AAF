# 阶段 5 执行任务清单 - 全局聊天与基础 WebSocket 架构

## 1. 当前状态

**✅ 已完成（阶段 5 收尾完成）** - 2026-03-13

本阶段目标：建立可鉴权、可广播、可持久化的实时聊天基础能力，作为后续世界内聊天与 VTT 实时事件的底座。

## 2. 已完成内容（本轮）

### 2.1 Socket 鉴权与连接基础

- 在 `server/src/socket/index.ts` 增加 Socket 连接鉴权：
  - 从 `handshake.auth.token` 或 `Authorization` 读取 JWT
  - 使用服务端 `JWT_SECRET` 验证
  - 将 `userId` 绑定到 `socket.data`

### 2.2 全局聊天事件

- 新增事件：
  - `global:message:send`（client -> server）
  - `global:message:new`（server -> client）
- 消息发送流程：
  1. socket 收到发送事件
  2. 服务端写入 `Message` 表（type=GLOBAL）
  3. 广播新消息到所有在线客户端

### 2.3 世界房间基础事件

- 新增 `world:join` 事件：
  - 校验用户是否为目标世界有效成员
  - 通过后加入 `world:{worldId}` room

### 2.4 聊天历史接口

- 新增 HTTP 接口：`GET /api/chat/global/recent`
- 支持 limit 参数（1-100）
- 返回按时间正序的最近全局消息

### 2.5 前端大厅实时聊天接入

- `client/src/lib/socket.ts`：
  - 动态读取服务端 host/port
  - 提供 `connectSocket(token)` 与 `disconnectSocket()`
- `client/src/pages/lobby/LobbyPage.tsx`：
  - 进入大厅后加载最近消息
  - 建立 socket 连接并监听新消息
  - 支持发送消息并处理 ack
