# 阶段 6 执行任务清单 - 最小 VTT Token 同步

## 1. 当前状态

**✅ 核心链路已完成（最小可用）** - 2026-03-13

本阶段目标：在已完成的世界房间与 WebSocket 基础上，落地最小 Token 实时同步能力。

## 2. 本轮完成内容

### 2.1 服务端 Socket 事件扩展

- 新增事件：
  - `scene:token:move`（client -> server）
  - `scene:token:moved`（server -> client）
- 新增最小 token 内存态缓存（按 worldId 分桶）。
- 用户加入世界房间后，可收到当前世界已存在 token 状态快照。

### 2.4 Token 状态持久化（增强）

- Token 状态已接入 `Scene.canvasState` 持久化。
- world join 时会先加载默认场景中的 token 状态。
- token move 时会写回默认场景，服务重启后可恢复。

### 2.5 多 Token 与基础权限（增强）

- 新增 token owner 字段（ownerUserId）用于最小权限判定。
- GM 可移动任意 token；非 GM 仅可移动自己的 token。
- 前端新增“新增我的 token”入口，默认以当前用户作为 owner。

### 2.2 世界房间生命周期接入

- 世界页在 socket 连接可用后自动发起 `world:join`。
- 世界页卸载时自动发起 `world:leave`。
- 实时监听 `world:members:update`，展示在线人数。

### 2.3 前端最小 VTT 交互

- 世界页新增 token 状态管理。
- `WorldCanvas` 从占位升级为可交互画板，支持点击移动 token。
- `TokenPanel` 新增最小控制能力（将主 token 置中）。

## 4. 验收要点

- 已登录用户进入世界页后可自动加入对应世界 room。
- 在任意客户端移动 token，其他同房间客户端能收到实时位置更新。
- 用户离开页面或断开连接后，在线人数会同步变化。
