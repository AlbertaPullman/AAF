## A. Hover Insight 系统 12px 间距 bug

### 现象

[client/src/world/components/HoverInsightCards.tsx](../client/src/world/components/HoverInsightCards.tsx) 第 45 行 `CARD_GAP = 12` 在锚点与卡之间留 12px 空隙；嵌套卡之间也有 12px。鼠标从锚点滑向卡片时如果鼠标轨迹穿过这 12px 空隙，DOM 的 `mouseenter` / `mouseleave` 触发链断裂，弹窗瞬间消失。

### 实测路径

锚点（如能力名 link）→ 用户鼠标移动 → 经过 12px 空隙 → 进入卡片 → 但因为中途经过非锚点非卡片的元素，hover 状态被取消 → 卡片关闭 → 用户没看到内容就走了。

### 修复方案（待实装）

三选一或组合：

1. **`mouseleave` 延迟**：卡片消失改为 200~300ms 延迟，期间鼠标进入卡片或锚点可取消消失。最简单、最有效。
2. **透明 hover 桥接层**：在锚点矩形与卡片矩形之间渲染一个透明 div 覆盖 12px 空隙，让该 div 也算 hover 范围。
3. **统一卡组 hover 状态**：用 React Context 把"任意锚点或任意卡正在 hover"作为整组的 active 状态，任一保持 = 整组保持。

推荐：先做 (1)，必要时再加 (2)。

### 优先级

**P2**。本期不做（不阻塞 P0/P1 工作）。哪一次手痒进 `HoverInsightCards.tsx` 时顺手修。

### 前置依赖

无。改的是组件内部状态机。
