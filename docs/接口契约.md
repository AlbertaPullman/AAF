# 接口契约

本文档提供当前 AAF 项目的 API 分组与调用约定，作为开发与联调的统一入口。

## 1. 通用规范

- 基础前缀：`/api`
- 认证方式：`Authorization: Bearer <token>`
- 响应结构：

```json
{
  "success": true,
  "data": {},
  "error": null,
  "requestId": "..."
}
```

## 2. 主要接口分组

### 2.1 世界与场景

- `GET /api/worlds`
- `POST /api/worlds`
- `GET /api/worlds/:worldId`
- `GET /api/worlds/:worldId/scenes`
- `POST /api/worlds/:worldId/scenes`
- `PATCH /api/worlds/:worldId/scenes/:sceneId/visual`
- `GET /api/worlds/:worldId/scenes/:sceneId/combat`
- `PUT /api/worlds/:worldId/scenes/:sceneId/combat`
- `POST /api/worlds/:worldId/scenes/:sceneId/combat/next-turn`

### 2.2 世界能力系统实体

- 能力：`/api/worlds/:worldId/abilities`
- 种族：`/api/worlds/:worldId/races`
- 职业：`/api/worlds/:worldId/professions`
- 背景：`/api/worlds/:worldId/backgrounds`
- 物品：`/api/worlds/:worldId/items`
- 命刻：`/api/worlds/:worldId/fate-clocks`
- 卡组：`/api/worlds/:worldId/decks`
- 随机表：`/api/worlds/:worldId/random-tables`

### 2.3 合集包导入导出

- 导出：`GET /api/worlds/:worldId/collection-pack`
- 导入：`POST /api/worlds/:worldId/collection-pack/import`

### 2.4 社交与聊天

- `GET /api/social/friends`
- `GET /api/social/requests/incoming`
- `POST /api/worlds/:worldId/chat-channels`
- `POST /api/worlds/:worldId/chat-channels/:channelKey/invite`

## 3. 权限原则

- GM：拥有世界核心配置与实体改写权限。
- GM 助手：可协助场景操作（不含核心规则改写）。
- 玩家：以游玩与角色操作为主。
- 旁观者：只读观察，不干预世界状态。

## 4. 推荐实践

1. 新增接口时同步更新 `docs/interface-registry.json`。
2. 跨端字段命名保持一致，不做前后端“别名映射”。
3. 高风险接口（导入、批量改写）必须保留明确权限校验。
