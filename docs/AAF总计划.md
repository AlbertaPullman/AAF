# AAF 总计划

更新时间：2026-04-17  
状态：执行中  
用途：本文件是项目唯一总计划，统一替代旧的阶段路线图、执行清单、验收清单与优化池文档。

## 1. 项目目标

AAF 是一个面向中文 TRPG 团务的 Web 平台。项目目标不是只做一个地图页面，而是提供完整的线上团务运行环境：

- 用户登录、进入大厅、创建和加入世界
- 在世界内完成聊天、场景切换、角色管理和规则交互
- 支持世界内运行时系统、战斗与结算扩展
- 为后续 AI 辅助、规则编辑器、数据包生态预留稳定结构

## 2. 当前基线

已稳定具备的能力：

- 账号注册、登录、鉴权
- 大厅与世界创建、加入
- 全局聊天与世界内聊天
- 多场景管理与基础隔离
- 角色基础 CRUD
- Socket.IO 实时同步基础链路
- 服务端自动化测试主集
- 前后端完整构建链路

当前已验证状态：

- `npm test -w server` 通过
- `npm run build` 通过

## 5. 当前问题清单

1. `client/src/pages/world/WorldPage.tsx` 体量过大，后续迭代成本高。
2. 部分历史中文文档存在乱码，已处理入口文档，但专题文档仍需分批清理。
3. 前后端个别字段命名仍有“历史命名”和“现行命名”并存的问题。

## 7. 验收口径

每项能力完成时，至少验证三类内容：

1. 正常路径：主流程是否可跑通
2. 权限边界：GM、玩家、观察者是否符合预期
3. 异常路径：缺参数、无权限、空状态、断线后是否能合理失败

项目层面的发布前验收至少包括：

1. 账号登录与大厅链路
2. 世界创建、加入、进入链路
3. 世界内聊天与实时同步链路
4. 场景、角色、战斗主链路
5. 构建与测试通过
