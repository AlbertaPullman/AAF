# 新会话接手引导（必读）

当开启新的 AI 对话继续开发本项目时，必须先执行以下步骤。

## 1. 必读顺序

1. 需求确认与填写清单.md
2. 渐进式开发文档.md
3. AI上下文记忆文档.md
4. docs/naming-convention.md
5. docs/interface-registry.json
6. 阶段1执行任务清单.md（若阶段1已完成则按后续阶段任务）

## 2. 开始编码前必须输出

1. 当前阶段判断（阶段0/1/2...）。
2. 本次目标范围（不超过一个阶段内的一个子目标）。
3. 涉及文件列表。
4. 验证命令列表。

## 3. 每次改动后必须同步

1. 更新 AI上下文记忆文档.md 的变更记录。
2. 若新增接口或事件，更新 docs/interface-registry.json。
3. 若新增命名规则，更新 docs/naming-convention.md。
4. 若调整阶段计划，更新 渐进式开发文档.md。

## 4. 禁止事项

1. 禁止跳过阶段直接实现后续复杂功能。
2. 禁止改动未在本次目标范围内的关键接口名。
3. 禁止把世界模块逻辑堆进单文件，必须组件化。
4. 禁止把酒馆模块与核心主链路强耦合。

## 5. 酒馆模块策略

1. 酒馆在目录上保持独立模块。
2. 若酒馆实现失败，可仅替换 tavern 模块为外部 API 适配层。
3. 主业务（账号、世界、聊天、VTT）不依赖酒馆可用性。
