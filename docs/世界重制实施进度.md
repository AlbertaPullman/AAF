## Loop 启动检查清单（每次 loop 起手必读）

1. **Read this file** → 拿到 `cursor`。
2. **Cursor == `ALL_P0_COMPLETE`?** → 不再推进，仅向用户简报最近一次完成情况，结束。
3. **Cursor 形如 `BLOCKED_AT_CP-NN`?** → 不动手，向用户报告阻塞原因，结束。
4. **Cursor == `CP-NN`?** → 打开 `docs/世界模板实现蓝图.md` §22 找到对应 CP 条目。
5. 按 CP 条目的「输入 / 动作 / 验证 / 完成标记」执行。
6. **每步动作前**：读取该 CP "输入" 列出的所有文件（这些文件可能在 loop 之间发生变化）。
7. **遇到非平凡决策时**：不要硬上。在「异常 / 阻塞记录」追加一条，cursor 改成 `BLOCKED_AT_CP-NN`，停下。
8. **完成后**：跑「验证」列出的所有命令。**全部通过**才在「完成清单」追加 `CP-NN ✅ ...`。
9. **任何验证失败**：保持 cursor 不变，在「异常 / 阻塞记录」追加失败原因 + 标记 `BLOCKED_AT_CP-NN`，停下。
10. **不 git commit**：所有 commit 等用户明确指令。
11. 同步 `AI上下文记忆文档.md` 加一行变更说明。
12. 向用户简报本轮做了什么 + 下次会做什么。
