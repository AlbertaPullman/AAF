# AAF 最终验收清单（完整版路线图）

更新时间：2026-03-15
状态：可执行清单（持续补全中）
使用方式：每完成一个子模块，按“操作步骤 -> 预期结果 -> 验收结论”执行并打勾。

## 0. 验收总原则

1. 所有验收以“真实链路可运行”为标准，不以代码存在为标准。
2. 每个功能至少覆盖：正常路径、权限边界、异常路径。
3. 先验收主链路，再验收增强功能。
4. 验收结果记录到本文件并同步 docs/change-log.md。

## 1. 环境与启动验收

### 1.1 依赖与环境变量

- [ ] 操作：执行 npm install。
- [ ] 操作：检查 .env 必填项（CLIENT_PORT、SERVER_PORT、DATABASE_URL、JWT_SECRET）。
- [ ] 预期：依赖安装成功且后端可读取环境变量。

### 1.2 一键启动脚本

- [ ] 操作：双击根目录 start.bat。
- [ ] 预期：自动打开两个终端（server/client）并自动打开 http://localhost:5174。
- [ ] 预期：前端页面可访问、后端日志无致命报错。

### 1.3 构建检查

- [ ] 操作：执行 npm run build -w server。
- [ ] 操作：执行 npm run build -w client。
- [ ] 预期：构建全部通过。

## 2. 平台主链路验收（阶段1-7）

### 2.1 认证链路

- [ ] 操作：注册 -> 登录 -> /api/auth/me。
- [ ] 预期：token 可用，鉴权状态持久化，401 时前端自动清理登录态。

### 2.2 世界链路

- [ ] 操作：创建世界 -> 加入世界 -> 进入世界。
- [ ] 预期：世界成员关系正确，角色权限正确显示。

### 2.3 聊天与实时

- [ ] 操作：大厅全局聊天发送与历史读取。
- [ ] 操作：世界聊天 OOC/IC/SYSTEM 收发与切换。
- [ ] 预期：频道权限生效（SYSTEM 仅 GM 可发），未读计数正确。

### 2.4 场景与角色卡

- [ ] 操作：场景创建、重命名、排序、删除（含边界：最后一个场景不可删）。
- [ ] 操作：角色创建、编辑、绑定 token。
- [ ] 预期：GM 与玩家权限边界正确。

### 2.5 最小VTT同步

- [ ] 操作：多端进入同一 world+scene，移动 token。
- [ ] 预期：scene 维度同步正确，不串场景。
- [ ] 预期：非 GM 不能移动他人 token。

## 3. 战斗与场景运行时验收（阶段8-B）

### 3.1 Runtime与模块管理

- [ ] 操作：查看 runtime 状态与模块列表。
- [ ] 操作：GM/ASSISTANT 启停模块，普通玩家尝试操作。
- [ ] 预期：权限与依赖校验正确。

### 3.2 视觉状态

- [ ] 操作：网格开关与单位调整。
- [ ] 操作：迷雾启停与模式切换。
- [ ] 操作：光源添加与清空。
- [ ] 预期：状态保存后刷新仍可读取。

### 3.3 战斗回合

- [ ] 操作：设置战斗状态、添加参战者、推进 next-turn。
- [ ] 预期：turnIndex 与 round 变化正确。
- [ ] 预期：无参战者时推进受限。

## 4. 剧情事件MVP验收（当前已开工）

### 4.1 GM 即兴事件

- [ ] 操作：GM 创建空白/即时事件（标题、描述）。
- [ ] 操作：GM 新增多个选项并配置 skillKey + DC + checkMode。
- [ ] 预期：玩家端可实时看到可见事件。

### 4.2 玩家检定提交

- [ ] 操作：玩家提交 finalTotal（可附 chatContent）。
- [ ] 预期：SINGLE / PER_PLAYER / UNLIMITED 次数规则生效。
- [ ] 预期：带 chatContent 时，聊天出现检定标签消息。

### 4.3 事件结算卡片

- [ ] 操作：GM 结算事件并填写 summary/finalOutcome。
- [ ] 预期：SYSTEM 频道自动出现事件结算卡片。
- [ ] 预期：聊天卡片能显示简述、经过、后果。
- [ ] 预期：/story-events/cards 可读取存档卡片。

### 4.4 权限与异常

- [ ] 操作：非 GM 尝试创建/编辑/结算事件。
- [ ] 预期：返回权限错误。
- [ ] 操作：提交非法检定值、无效选项ID。
- [ ] 预期：返回参数错误且不污染数据。
