---

## 1. 迭代 A 范围冻结

### 1.1 本迭代必须完成

1. 世界运行时状态管理（World Runtime State）。
2. 模块注册机制（注册、启停、依赖检查、懒加载入口）。
3. 事件总线基础能力（发布、订阅、退订、命名约束）。
4. 中文文案基础框架（世界内模块统一中文文案入口）。

### 1.2 本迭代禁止扩展

1. 不实现战斗规则细节（先攻、伤害、反应结算逻辑后置到迭代 B/C）。
2. 不实现完整 AI 聊天链路（后置到迭代 E）。
3. 不实现完整可视化编辑器（后置到迭代 D）。

---

## 2. 任务泳道

## 2.1 后端泳道（P0）

### A-BE-01 世界运行时状态模型（已完成）

产出：

1. 世界运行时状态结构定义（active/sleeping/loading/error）。
2. 基础服务：创建状态、更新状态、读取状态。

建议文件：

1. server/src/modules/world-runtime/world-runtime.types.ts
2. server/src/modules/world-runtime/world-runtime.service.ts

验收：

1. 可按 worldId 获取运行状态。
2. 状态更新具备基础参数校验。

### A-BE-02 模块注册中心（已完成）

产出：

1. ModuleRegistry（register/enable/disable/get/list）。
2. 依赖校验机制（缺失依赖不可启用）。
3. 懒加载钩子入口（先提供接口，不要求完成所有模块接入）。

建议文件：

1. server/src/modules/runtime/module-registry.ts
2. server/src/modules/runtime/module.types.ts

验收：

1. 模块可注册并可查询状态。
2. 非法依赖会返回明确错误。

### A-BE-03 事件总线基础（已完成）

产出：

1. Typed Event Bus（on/off/emit/once）。
2. 事件名校验（domain:entity:action）。
3. 统一错误结构。

建议文件：

1. server/src/modules/runtime/event-bus.ts
2. server/src/modules/runtime/event-naming.ts

验收：

1. 可完成最小事件发布与订阅。
2. 非法事件名会被拒绝并给出中文提示。

### A-BE-04 运行时管理接口（已完成）

产出：

1. 查询运行时状态接口。
2. 查询已注册模块接口。
3. 启停模块接口（仅 GM 或更高权限）。

建议文件：

1. server/src/controllers/world-runtime.controller.ts
2. server/src/routes/world-runtime.routes.ts
3. server/src/routes/world.routes.ts（挂载子路由）

验收：

1. 能在 API 层看到 world runtime 与 module 列表。
2. 权限不足时返回标准错误。
