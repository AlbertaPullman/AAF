## 1. 文件夹数据模型（影响所有面板）

### Q1-1 文件夹存储方式

现有资源（能力/物品/职业/种族/背景/命刻/牌组/随机表）已有 `folderPath?: string` 字段，FVTT 用的是独立 Folder 表。两种走法：

- **A. 继续用 `folderPath` 字符串**（如 `卡拉塔纳/危险生物/精魂纳迦`），开发量小，迁移容易，但**不支持**同名文件夹的复杂权限/排序/图标。

## 2. 场景管理（FVTT 化）细节

### Q2-1 缩略图来源

FVTT 的缩略图是把背景图直接缩小渲染。我们的场景已存 `backgroundImageUrl`，我打算：

- 上传背景图时**服务端生成 256×256 缩略图**存到对象存储，字段 `thumbnailUrl`；
- 没背景图就显示场景类型图标（战斗/探索/休整等）+ 渐变色块兜底。

**Q2-1**：服务端生成缩略图 OK 吗？还是前端上传时本地生成 dataUrl 直接存？后者省服务器但占 DB 空间。  
> _你的答案_：省服务器资源的，服务器硬盘空间足够，但是算力有点差，所以要优化好。

### Q2-2 场景"激活"语义

你的图 2 右键菜单里有"查看场景"和"激活"两个动作。我的理解：

- **查看场景**：打开当前用户的舞台预览（不影响其他玩家）。

**Q2-2**：这个理解对吗？如果错，告诉我两者的真实区别。  
> _你的答案_：对，"查看场景"只影响当前用户，"激活"会把所有玩家拉到该场景。

## 4. HUD ↔ 角色卡绑定

### Q4-1 HUD 配置存在哪

你说"角色卡应该会跟 HUD 面板的配置绑定"。两种存法：

- **A. HUD 配置随角色卡（每个角色卡有自己的 `hudConfig`）**——怪物 HUD 永远跟卡走，玩家可以自调。
- **B. HUD 配置按"模板"绑定**（角色卡 → 模板 ID → HUD 配置）——多个相同怪物共用同一份。

**Q4-1**：A 还是 B？  
> _你的答案_：按照B共用同一份，但是可以把单个怪物的HUD拆出来单独做。默认是使用模板，用模板的话需要给怪物的动作分类，按照分类存放在HUD的不同位置。

### Q4-2 GM 切 token 时 HUD 行为

你说"GM 选中哪个角色的 token，HUD 上面展示的就是对应角色的面板"。细节：

1. GM 同时选中多个 token（框选）时，HUD 显示什么？最后一个？空？多选模式？
2. GM 选中**怪物**时，HUD 是否显示怪物完整数据（包括隐私字段）？
3. GM 没选任何 token 时，HUD 显示什么？空？还是"GM 的默认角色卡"（如果 GM 也有自己的角色）？

**Q4-2**：分三句答。  
> _你的答案_：同时选中多个token时，HUD为空。选中怪物时，HUD显示怪物完整数据。没选任何token时，HUD显示GM的默认角色卡。（如果GM有自己的角色，如果没有就是空。）

### Q4-3 玩家选择敌方 token

玩家点击敌方 token 时，HUD 是否会切换？还是玩家 HUD 永远固定显示自己的角色？我倾向后者（避免误操作 + 信息不串）。

**Q4-3**：玩家 HUD 固定 = A；玩家 HUD 也跟选中切换 = B。  
> _你的答案_：玩家 HUD 永远固定显示自己的角色，避免误操作和信息串。除非玩家切换到自己的部署上，此时显示的是部署的HUD。

### Q5-1 "部署单位"是什么

候选解释：

- **A. 召唤物 / 战宠**：玩家用法术/能力召唤的临时单位（持续多回合）。

> _你的答案_：物语点无上限，游戏开始时（玩家刚创建角色时）获得3点，随后抵达满足条件时获得：
获得物语点
玩家角色有四种获得物语点的主要方法：
·如果玩家角色在一次新聚会开始时没有物语点，她们将立即获得1个物语点
·每当玩家角色在检定中出现大失败时，她立即获得1物语点

角色的物语点没有上限。简而言之，你可以为了“重要的场合”而随意储存它们。
