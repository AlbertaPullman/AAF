# 阶段 1 执行任务清单（文件级）

本清单用于直接开始阶段 1：项目骨架初始化。目标是搭出可运行、可扩展、可继续分阶段开发的基础工程。

## 1. 阶段目标

1. 建立前后端基础目录与运行脚本。
2. 确立统一工程规范（环境变量、返回格式、错误处理、日志）。
3. 建立 Socket 服务入口但不写复杂业务事件。
4. 为阶段 2 的 Prisma 与数据库建模预留结构。

## 2. 建议目录结构


## 3. 文件级任务

### 3.1 根目录文件

1. 新建 package.json
   目的：统一 workspace 脚本，例如 dev、build、lint。
2. 新建 .gitignore
   目的：忽略 node_modules、构建产物、日志、数据库文件和本地环境变量。
3. 新建 .editorconfig
   目的：统一缩进与换行，避免多人协作格式漂移。
4. 新建 README.md
   目的：写清启动步骤、目录说明、环境变量说明。

## 4. 阶段 1 验收清单

1. 前端和后端能分别独立启动。
2. 根目录命令可一键并发启动前后端。
3. 访问健康检查接口返回统一格式。
4. Socket 连接成功时有日志输出。
5. 文档中有清晰运行说明。

## 5. 阶段 1 前置确认（必须先拍板）

1. 已完成，无阻塞项。

## 6. 模块化约束（强制）

1. 世界内功能按组件拆分目录，例如 token、measure、draw、scene、chat-panel。
2. 每个组件独立维护事件与状态，不允许把所有画布逻辑堆在单文件中。
3. 酒馆功能单独模块，不与世界主逻辑耦合，保证可禁用、可替换。

## 7. 开工顺序建议

1. 先建根目录文件与脚本。
2. 再建 server 最小可运行。
3. 再建 client 最小可运行。
4. 再接 shared 常量。
5. 最后补 docs 协议文档。

这个顺序可以保证每一步都可执行、可验证，不会出现“目录建完了但跑不起来”的空壳状态。
